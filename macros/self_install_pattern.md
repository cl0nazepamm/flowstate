# CloneTools Self-Install Pattern for MAXScript

How CloneTools macroscripts register themselves for 3ds Max menu/toolbar access without requiring drag-and-drop.

## The Problem

3ds Max only auto-registers macroscripts from `.mcr` files in the `userMacros` directory. If your script lives elsewhere (a repo, a network share, a zip), the user has to manually drag it into the viewport — and that registration doesn't survive if the file moves.

## The Solution

Ship a single `.ms` file that contains:

1. **Global variable** — `g_CloneTools_ScriptsDir`
2. **Install helper functions** — writable-dir check, path escaping
3. **Startup loader installer** — `FlowState_InstallStartupLoader()` (or `AddUSDProps_InstallStartupLoader()`)
4. **The macroscript block** — `macroscript ... ( ... )` works fine inside a `.ms`
5. **Self-install call at file scope** — runs when the file is `fileIn`'d or dragged

## How Each Piece Works

### 1. Global: `g_CloneTools_ScriptsDir`

```maxscript
global g_CloneTools_ScriptsDir
```

Set at file scope when the script first runs. Points to the directory the `.ms` lives in. Other scripts in the pipeline can use this global to find companion files (attribute definitions, icons, etc.) without hardcoding paths.

Set via `getSourceFileName()` at the bottom of the file:

```maxscript
(
    local selfPath = getSourceFileName()
    if selfPath != undefined and selfPath != "" do
        ::g_CloneTools_ScriptsDir = getFilenamePath selfPath
)
```

### 2. Copy to `userMacros`

```maxscript
local userMacrosDir = getDir #userMacros
-- copy self to userMacrosDir
```

3ds Max scans `userMacros` at startup and registers any `macroscript` blocks it finds. By copying the `.ms` there, the macroscript appears in **Customize User Interface → Category: CloneTools** permanently. The user can then assign it to a menu, toolbar, or quad menu.

### 3. Copy to `userScripts\CloneTools\` (fallback)

```maxscript
local targetDir = (getDir #userScripts) + "\\CloneTools\\"
```

A second copy goes here as a fallback location. If the original source moves (repo relocated, USB removed), the startup loader can still find this copy.

### 4. Startup Loader

The installer writes a small `.ms` file into startup directories:

```
userStartupScripts\FlowState_LoadMacros.ms
```

Contents (auto-generated):

```maxscript
-- Auto-generated by flowstate
(
    local macroPaths = #(
        "C:\\original\\path\\flowstate_config.ms",
        "C:\\Users\\...\\userScripts\\CloneTools\\flowstate_config.ms"
    )
    for p in macroPaths do (
        if doesFileExist p do (
            fileIn p quiet:true
            exit
        )
    )
)
```

This runs every time 3ds Max starts. It tries the original source path first (so edits in the repo are picked up immediately), then falls back to the installed copy. The `fileIn` re-executes the whole `.ms`, which:
- Re-sets the global
- Re-registers the macroscript (with any updates)
- Re-runs the installer (copies updated file to userMacros/userScripts)

### 5. Writable Directory Check

```maxscript
fn FlowState_isWritableDir dirPath = (
    -- creates a temp file, deletes it, returns true/false
)
```

Before writing the startup loader, the installer probes each startup directory. Some may be read-only (locked-down studio environments, system-level startup dirs). Only directories that pass the write test get a loader installed.

### 6. Path Escaping

```maxscript
fn FlowState_escapePath path = (substituteString path "\\" "\\\\")
```

The startup loader is written as a string and then `format`'d to a file. Backslashes in Windows paths need double-escaping so they survive the string→file round-trip.

## File Layout

```
flowstate_config.ms
├── global g_CloneTools_ScriptsDir
├── fn FlowState_isWritableDir ...
├── fn FlowState_escapePath ...
├── fn FlowState_InstallStartupLoader ...
│   ├── copy self → userScripts\CloneTools\
│   ├── copy self → userMacros\
│   └── write FlowState_LoadMacros.ms → startup dirs
├── macroscript flowstate_Config category:"CloneTools" ( ... )
├── (set global from getSourceFileName)
└── FlowState_InstallStartupLoader()
```

## Flow: First Run

1. User runs `fileIn "flowstate_config.ms"` or drags it into Max
2. Helper functions are defined
3. `macroscript flowstate_Config` block is parsed → registered immediately
4. `g_CloneTools_ScriptsDir` is set to the source directory
5. `FlowState_InstallStartupLoader()` runs:
   - Copies `.ms` to `userScripts\CloneTools\` and `userMacros\`
   - Writes `FlowState_LoadMacros.ms` into startup directories
6. Macroscript is now available in Customize UI under **CloneTools**

## Flow: Subsequent Startups

1. 3ds Max starts, scans startup directories
2. `FlowState_LoadMacros.ms` runs
3. It `fileIn`s the `.ms` from the first available path
4. Macroscript is re-registered, global is set, installer re-copies (keeps files in sync)

## Working Examples

- `flowstate\macros\flowstate_config.ms` — config dialog
- `powerusdexporter\addusdproperties.ms` — USD property tool (same pattern, more dependencies)
